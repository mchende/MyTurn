## 整节课节奏编排

| Option | Description | Selected |
|--------|-------------|----------|
| 四段式课堂 | `开场热身 -> 复述主段 -> 看图主段 -> 结课收束`，课堂感最完整，也最像真实短课 | ✓ |
| 三段式直推进 | `开场 -> 混合练习 -> 结课`，更紧凑，但逐步升级感更弱 | |
| 两轮循环式 | 先跑一小轮复述+看图，再重复第二轮，更游戏化 | |

**User's choice:** 四段式课堂  
**Notes:** 用户希望完整 lesson 更像一节短课，而不是几个练习拼起来。

| Option | Description | Selected |
|--------|-------------|----------|
| 后半程更重 | 把主要时间留给 picture-talk 和主动输出 | |
| 前后均衡 | 复述和看图时长差不多，整体更平稳 | ✓ |
| 前半程更重 | 更多时间给复述建立信心，看图只轻量收口 | |

**User's choice:** 前后均衡  
**Notes:** 复述和看图都要像正式主段，不希望任何一段沦为过渡。

| Option | Description | Selected |
|--------|-------------|----------|
| 老师带班开场 | 先有 greeting、点名和入课状态，再进第一题 | ✓ |
| 直接进第一题 | 更省时，但起课感弱 | |
| 老师+Bobby 小互动开场 | 氛围更活，但会吃掉题目时间 | |

**User's choice:** 老师带班开场  
**Notes:** 用户明确希望先有“真的上课了”的入课感。

| Option | Description | Selected |
|--------|-------------|----------|
| 老师口头收束 + 明确下课状态 | 由老师说 goodbye / praise，并进入完成态 | ✓ |
| reward 后直接停在课堂页 | 实现最省，但下课感更弱 | |
| 老师收束后直接返回首页 | 网页闭环更强，但更像流程跳转 | |

**User's choice:** 老师口头收束 + 明确下课状态  
**Notes:** 结课需要像一节课真正结束，而不是流程结束。

## 后半程看图作答强度

| Option | Description | Selected |
|--------|-------------|----------|
| 回答老师的问题 | 老师基于图片发问，孩子回答，最像课堂问答 | ✓ |
| 用词/短句描述图里内容 | 更开放，但更像看图说词 | |
| 两者混合 | 变化更丰富，但第一版容易做散 | |

**User's choice:** 回答老师的问题  
**Notes:** 用户希望继续保持老师带班主导的问答节奏。

| Option | Description | Selected |
|--------|-------------|----------|
| 同一张图先简单问再展开 | 升级更自然，后期输出更主动 | |
| 每张图都只问一个简单问题 | 最稳，更适合先把完整课跑顺 | ✓ |
| 一上来就要求完整句回答 | 主动输出更强，但风险更高 | |

**User's choice:** 每张图都只问一个简单问题  
**Notes:** 用户优先考虑完整 lesson 的稳定和自然感，不在 Phase 5 再明显提难度。

| Option | Description | Selected |
|--------|-------------|----------|
| 继续课堂节奏，不降级回复述 | 最像真实课堂，不把课打回前半程 | ✓ |
| 临时插回一小段更容易的跟说 | 更照顾信心，但结构会变松 | |
| 提前进入收束 | 最安全，但完整课感更弱 | |

**User's choice:** 继续课堂节奏，不降级回复述  
**Notes:** 即使多题 fallback，也不要把课堂改造成自适应补救流程。

| Option | Description | Selected |
|--------|-------------|----------|
| 很短的课堂式肯定，然后立刻下一题 | 节奏干净，像小班课 | ✓ |
| 每题都给更明显的奖励感 | 更热闹，但容易像闯关 | |
| 大部分不反馈，只靠推进 | 更利落，但温度可能不够 | |

**User's choice:** 很短的课堂式肯定，然后立刻下一题  
**Notes:** 用户希望保留课堂温度，但不希望奖励感打断 lesson pacing。

## 结课与完成态

| Option | Description | Selected |
|--------|-------------|----------|
| 只在整节课结束时出现一次完整庆祝 | 结尾统一给完成感，最像下课表扬 | ✓ |
| 中间偶尔有小庆祝，结尾再来一次 | 更热闹，但容易打断课堂 | |
| 不做明显 reward，只靠老师口头表扬 | 更像真实课，但网页闭环弱 | |

**User's choice:** 只在整节课结束时出现一次完整庆祝  
**Notes:** 用户要把 reward 集中到整节课结束，不在中途分散课堂注意力。

| Option | Description | Selected |
|--------|-------------|----------|
| 完成卡片留在课堂页 | 最像刚下课、还在教室里 | ✓ |
| 自动回到首页 | 闭环更强，但像跳流程 | |
| 弹出继续按钮，由孩子决定回首页 | 控制感更强，但更工具化 | |

**User's choice:** 完成卡片留在课堂页，并在约 3 秒后自动跳回首页  
**Notes:** 用户先选了课堂页完成态，又追加了“3 秒后自动跳首页”的具体收束要求。

## 网页端收口与入口/视口问题

| Option | Description | Selected |
|--------|-------------|----------|
| 改成响应式重排 | 窄屏时改为更稳的重排布局，保证内容都可见 | ✓ |
| 坚持大屏布局，只做最小缩放 | 保住当前视觉，但小窗口风险仍高 | |
| 窄屏时切成简化版首页 | 最稳，但会变成两套首页 | |

**User's choice:** 改成响应式重排  
**Notes:** 用户明确提到首页中部和右侧在某些视口下“不显示”，因此希望 Phase 5 一并收口这个问题。

| Option | Description | Selected |
|--------|-------------|----------|
| 显示已完成状态，并保留完成余温 | 回首页后一眼能看出这节刚上完 | ✓ |
| 恢复普通首页，不特别标记 | 最简单，但闭环感弱 | |
| 显示更明显的课后总结卡 | 更强反馈，但范围变大 | |

**User's choice:** 显示已完成状态，并保留完成余温  
**Notes:** 用户不想让自动返回首页后看起来“什么都没发生”。
