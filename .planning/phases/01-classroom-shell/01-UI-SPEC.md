## Phase Intent

- 本 phase 的视觉目标不是做学习工具 dashboard，而是让孩子一打开页面就感到“今天有课，而且正在准备进入一节正式的小班课”。
- 首页只服务三件事：看今日课表、判断当前是否可入场、找到下一节课。
- 课堂页先建立稳定空间关系，不提前承载语音、评分、复杂动效或强交互。
- 平板横屏是主设计目标；手机与桌面端做结构保真缩放，不改变信息层级。

## Layout Contract

### Schedule Surface

- 首页采用“今日课表”单核心视图，不出现与上课无关的管理信息、成就信息或学习统计。
- 页面首屏包含三层信息：
  - 顶部状态区：今日日期、课堂品牌标题、当前时间语义提示。
  - 中部主内容区：完整今日课表列表，按时间顺序排列。
  - 底部辅助区：下一节课提示或错过当前课次后的引导提示。
- 当前最相关课次必须在视觉上被突出，但不得隐藏全天其他课次。
- 每个课次卡片固定包含：开始时间、结束时间、课时长度、状态标签、主 CTA 或锁定提示。

### Classroom Surface

- 课堂页固定采用三段式课堂空间：
  - 顶部学生席
  - 左侧课件主屏
  - 右侧讲台区与教师区
- 左侧课件主屏为绝对视觉主角，占桌面/平板横屏宽度约 60%。
- 右侧角色区占约 40%，内部再分为上方讲台区、下方教师区，推荐比例 5:4。
- 顶部学生席横向铺开，显示真实孩子、AI 同学和 2 个空座位氛围位，共 4 席。
- 讲台区默认可见，即使尚未点名也显示“待上台”状态，不允许空白塌陷。
- 教师区固定钉住右下区域，不随课件切换或学生状态变化而移位。

### Responsive Rules

- 平板横屏断点为主断点，主布局保持左右分栏和顶部学生席完整存在。
- 手机端将顶部学生席压缩为可横向滚动或两列紧凑排列，但仍需保留“从席位到讲台”的关系表达。
- 手机端右侧讲台区与教师区可改为纵向堆叠，但教师区必须持续可见。
- 桌面端允许增加外边距与留白，不增加新的功能面板。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 图标与数字微间距、状态点间隔 |
| sm | 8px | 卡片内部紧凑间距、标签与文本间隔 |
| md | 16px | 默认组件内边距、正文块间距 |
| lg | 24px | 卡片内主要分区、课表项垂直节奏 |
| xl | 32px | 页面主区块间距、课堂分栏外间距 |
| 2xl | 48px | 首页首屏上下留白、课堂主舞台呼吸感 |
| 3xl | 64px | 平板与桌面大屏页面边距、场景级间隔 |

Exceptions: 触控目标最小高度 44px；课表主 CTA 高度 48px；顶部学生席头像点击/聚焦区域最小 44px。

### Spacing Rules

- 课表卡片内部使用 `md` 作为基础 padding，当前推荐课次可提升到 `lg`。
- 课堂主屏、讲台区、教师区都使用圆角大容器，但容器之间的主间距统一用 `xl`。
- 空座位氛围位之间的距离不得大于真实席位间距，避免显得像独立功能按钮。
- 所有状态标签与正文之间优先使用 `sm`，不使用 6px、10px 这类非 token 间距。

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

### Type Rules

- 全站只使用两档字重：`400` 与 `600`。
- 中文正文、辅助说明、状态解释使用 `Body`。
- 时间、状态标签、席位名牌、次级按钮文本使用 `Label`。
- 课表分组标题、课堂区域标题、空状态标题使用 `Heading`。
- 首页主标题、课堂页课程名称、倒计时核心数字可使用 `Display`。
- 不使用全大写英文标签；英文仅用于文档结构、技术字段和少量产品识别词。

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F6F1E8` | 页面背景、课堂墙面、大面积底色 |
| Secondary (30%) | `#E6D6BF` | 课表卡片、讲台背景、教师区容器、顶部学生席底板 |
| Accent (10%) | `#F28C28` | 当前可入场课次、倒计时高亮、主 CTA、当前被点名席位描边 |
| Destructive | `#C9493C` | 仅保留给未来 destructive actions；本 phase 不实际使用 |

Accent reserved for: 当前可入场的课次卡片边框或底色强调、首页主 CTA、下一节课倒计时数字、课堂中当前上台学生的讲台高亮、当前被老师点名的席位状态。不得用于普通次级按钮、普通图标、装饰性背景或所有可点击元素。

### Color Rules

- 整体观感应接近温暖教室与木质讲台，而不是冷色 SaaS 控制台。
- Dominant 色占据页面背景和最大块面，确保课堂空间先被感知，再看到控件。
- Secondary 色用于承载结构层，帮助区分卡片、席位、讲台、教师区。
- Accent 色只给“现在该看哪里、该点哪里、谁正在被关注”。
- 锁定入场状态不使用 destructive 红作为主色，而使用去饱和深棕或灰棕文本加弱底色表达“已开课，停止入场”。

## Interaction Contract

### Schedule States

- `upcoming`: 显示开始时间和倒计时，不显示可点击主 CTA。
- `open_for_entry`: 课次卡片必须成为当前主焦点，显示主 CTA。
- `in_progress_locked`: 明确显示“已开课，停止入场”，CTA 禁用或替换为非交互文本。
- `completed`: 显示“已结束”或弱化历史状态，不再鼓励回看。

### Classroom States

- 进入课堂后，顶部学生席、课件区、讲台区、教师区必须同时存在，不能靠后续交互逐步出现。
- 学生从席位到讲台的状态变化应通过位置、边框、底色或轻微位移表达，不依赖复杂动画。
- 默认未点名状态下，讲台区显示“待上台”或“即将轮到谁”的占位信息。
- 课件图切换时保持容器尺寸稳定，避免课堂主屏抖动。

### Motion

- 动效总原则：服务课堂节奏，不服务炫技。
- 推荐只使用三类动效：
  - 页面进入时 200ms 到 300ms 的淡入上移
  - 当前课次或当前学生状态变化时 150ms 到 200ms 的强调过渡
  - 倒计时数字每秒刷新时无跳跃式 reflow
- 禁止大幅弹跳、旋转、连续漂浮、过度缩放。

### Accessibility

- 所有主操作必须可通过键盘聚焦。
- 颜色不是唯一状态来源；状态还需配合文案或图标。
- CTA、课次卡片、学生席位的交互目标不低于 44px。
- 课堂主屏图片必须有与课程目标相关的替代文本。

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 进入课堂 |
| Empty state heading | 今天暂时没有可进入的课程 |
| Empty state body | 请查看下一节课开始时间，并在开课前进入课堂。 |
| Error state | 今日课表加载失败。请刷新页面，或稍后再试。 |
| Destructive confirmation | 不适用：Phase 1 无 destructive actions |

### Copy Rules

- 用户可见文案默认使用中文，以降低儿童与陪同家长的理解门槛。
- 状态文案要像“上课安排提示”，不像系统报错或后台状态码。
- 推荐课次相关文案优先使用明确时间表达，例如“09:55 开始入场”或“10:00 正式开课”。
- 锁定状态统一使用“已开课，停止入场”，不使用“已失效”“不可用”“权限不足”。
- 下一节课提示必须出现具体时间，不写模糊表达如“稍后开始”。
