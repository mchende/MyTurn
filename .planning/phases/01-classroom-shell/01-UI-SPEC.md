## Phase Intent

- 本 phase 的视觉目标不是做学习工具 dashboard，而是让孩子一打开页面就感到“今天有课，而且正在准备进入一节正式的小班课”。
- 首页只服务三件事：看今日课表、判断当前是否可入场、找到下一节课。
- 课堂页先建立稳定空间关系，不提前承载语音、评分、复杂动效或强交互。
- 平板横屏是主设计目标；手机与桌面端做结构保真缩放，不改变信息层级。
- 整体视觉语言以 Stitch 原型定义的 Academic Sanctuary 方向为准：明亮底色、轻卡片、克制绿强调、温和圆角和明确层级，仍保持“课堂优先而非后台工具”的主基调。

## Layout Contract

### Schedule Surface

- 首页在平板与桌面端采用“两区布局”：左侧轻导航栏，右侧主内容区。
- 左侧导航栏至少包含品牌 `MyTurn`、导航项 `主页`、导航项 `设置`；`设置` 在 Phase 1 可为占位导航，不要求真正实现设置页路由。
- 右侧主内容区主标题使用 `我的课堂`，下方展示“今日全部课次”的响应式卡片网格，而不是统计 dashboard。
- 课次卡片按当天时间顺序排列；移动端为单列，平板推荐双列，较宽桌面可扩展到三列。
- 每个课次卡片固定包含：课程标题、开始时间、课时长度、班级/学生信息、状态提示、主 CTA 或倒计时。
- 当前最相关课次必须在视觉上被突出，但不得隐藏全天其他课次。
- upcoming 课次优先展示“距上课 xx 分”和大号开始时间；open_for_entry 课次展示主 CTA；in_progress_locked 课次展示锁定文案。

### Classroom Surface

- 课堂页固定采用三段式课堂空间：
  - 顶部学生席
  - 左侧课件主屏
  - 右侧讲台区与教师区
- 左侧课件主屏为绝对视觉主角，采用深色沉浸式舞台容器，占桌面/平板横屏宽度约 60% 到 70%。
- 右侧角色区占约 30% 到 40%，内部再分为上方讲台区、下方教师区，推荐比例接近 1:1 或 5:4。
- 顶部学生席横向铺开，显示真实孩子、AI 同学和 2 个空座位氛围位，共 4 席；当前上台学生可在顶部席位和右侧讲台卡片中双重强调。
- 讲台区默认可见，即使尚未点名也显示“待上台”状态，不允许空白塌陷；其视觉形式可贴近原型中的 active student card。
- 教师区固定钉住右下区域，不随课件切换或学生状态变化而移位。
- 课件主屏底部允许存在轻量控制条，用于页码、前后翻页或课堂控制，不增加额外业务面板。

### Responsive Rules

- 平板横屏断点为主断点：首页保留左导航 + 右内容区，课堂页保持顶部学生席与左右分栏完整存在。
- 手机端首页可将左导航折叠为顶栏或抽屉，但仍需保留 `主页` / `设置` 的访问入口，并保持课程卡片顺序不变。
- 手机端将顶部学生席压缩为可横向滚动或两列紧凑排列，但仍需保留“从席位到讲台”的关系表达。
- 手机端右侧讲台区与教师区可改为纵向堆叠，但教师区必须持续可见。
- 桌面端允许增加外边距与留白，不增加新的功能面板。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 图标与数字微间距、状态点间隔 |
| sm | 8px | 卡片内部紧凑间距、标签与文本间隔 |
| md | 16px | 默认组件内边距、正文块间距 |
| lg | 24px | 卡片内主要分区、课表项垂直节奏 |
| xl | 32px | 页面主区块间距、课堂分栏外间距 |
| 2xl | 48px | 首页首屏上下留白、课堂主舞台呼吸感 |
| 3xl | 64px | 平板与桌面大屏页面边距、场景级间隔 |

Exceptions: 触控目标最小高度 44px；课表主 CTA 高度 48px；顶部学生席头像点击/聚焦区域最小 44px。

### Spacing Rules

- 课表卡片内部使用 `md` 作为基础 padding，当前推荐课次可提升到 `lg`。
- 课堂主屏、讲台区、教师区都使用圆角大容器，但容器之间的主间距统一用 `xl`。
- 空座位氛围位之间的距离不得大于真实席位间距，避免显得像独立功能按钮。
- 所有状态标签与正文之间优先使用 `sm`，不使用 6px、10px 这类非 token 间距。

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.50 |
| Label | 14px | 600 | 1.29 |
| Heading | 32px | 700 | 1.20 |
| Display | 40px | 700 | 1.10 |

### Type Rules

- 全站优先使用 `400`、`600`、`700` 三档字重；`700` 主要用于页面主标题和课程卡片标题。
- 中文正文、辅助说明、状态解释使用 `Body`，并遵循偏紧的字距与高可读节奏。
- 时间、状态标签、席位名牌、次级按钮文本使用 `Label`。
- 课表分组标题、课堂区域标题、空状态标题使用 `Heading`。
- 首页主标题、课堂页课程名称、关键课堂标题使用 `Display`。
- 不使用全大写英文标签；英文仅用于文档结构、技术字段和少量产品识别词。
- Display 与 Heading 优先使用 Plus Jakarta Sans 语气，正文与标签优先使用 Manrope 语气；若环境不具备原字体，则选择最接近的 system fallback。
- 标题保持紧凑但不压缩成 Apple 式极限行高；正文行高保持舒适，贴近原型中轻量 SaaS 卡片界面的阅读节奏。

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `#F8FAFC` | 首页和后台承载底色、卡片外画布 |
| Secondary (30%) | `#111827` / `#E5E7EB` | 主文字、轻边框、深色课堂舞台、弱化态信息 |
| Accent (10%) | `#28C76F` / `#006A36` | 主 CTA、live 状态、可入场课次、当前活跃学生强调 |
| Destructive | `#EF4444` | 离开课堂等破坏性动作；仅课堂控制区可使用 |

Accent reserved for: 首页主 CTA、可入场课次强调、文本链接、focus 状态、课堂中当前被点名的学生状态。不得用于普通容器背景、普通次级按钮、装饰性图形、所有可点击元素的统一着色。

### Color Rules

- 整体观感以 Stitch 原型为准：首页使用浅底卡片世界，课堂页使用浅外框 + 深色主舞台的组合，而不是全站二元黑白 section 交替。
- Dominant 色负责场景气氛：浅色区域用于课表与信息，深色舞台仅留给课堂主屏。
- Secondary 色用于承载结构层、轻边框和主要文字层。
- Accent 色只给“现在该看哪里、该点哪里、谁正在被关注”，主色从 Apple Blue 切换为原型中的课堂绿。
- 锁定入场状态不使用 destructive 红作为主色，而采用 near-black/gray 的弱化处理表达“已开课，停止入场”。
- 首页卡片允许轻阴影和轻边框；课堂主舞台允许深色渐层或柔和背景图，但不得喧宾夺主。

## Interaction Contract

### Schedule States

- `upcoming`: 显示开始时间和倒计时，不显示可点击主 CTA。
- `open_for_entry`: 课次卡片必须成为当前主焦点，显示主 CTA。
- `in_progress_locked`: 明确显示“已开课，停止入场”，CTA 禁用或替换为非交互文本。
- `completed`: 显示“已结束”或弱化历史状态，不再鼓励回看。

### Classroom States

- 进入课堂后，顶部学生席、课件区、讲台区、教师区必须同时存在，不能靠后续交互逐步出现。
- 学生从席位到讲台的状态变化应通过位置、边框、底色或轻微位移表达，不依赖复杂动画。
- 默认未点名状态下，讲台区显示“待上台”或“即将轮到谁”的占位信息。
- 课件图切换时保持容器尺寸稳定，避免课堂主屏抖动。

### Motion

- 动效总原则：服务课堂节奏，不服务炫技。
- 推荐只使用三类动效：
  - 页面进入时 200ms 到 300ms 的淡入上移
  - 当前课次或当前学生状态变化时 150ms 到 200ms 的强调过渡
  - 倒计时数字每秒刷新时无跳跃式 reflow
- 禁止大幅弹跳、旋转、连续漂浮、过度缩放。
- 不使用夸张的儿童化反馈动效；遵循 `DESIGN.md` 所强调的克制、精确和“界面退后”原则。

### Accessibility

- 所有主操作必须可通过键盘聚焦。
- 颜色不是唯一状态来源；状态还需配合文案或图标。
- CTA、课次卡片、学生席位的交互目标不低于 44px。
- 课堂主屏图片必须有与课程目标相关的替代文本。

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page heading | 我的课堂 |
| Primary CTA | 进入课堂 |
| Empty state heading | 今天暂时没有可进入的课程 |
| Empty state body | 请查看下一节课开始时间，并在开课前进入课堂。 |
| Error state | 今日课表加载失败。请刷新页面，或稍后再试。 |
| Destructive confirmation | 不适用：Phase 1 无 destructive actions |

### Copy Rules

- 用户可见文案默认使用中文，以降低儿童与陪同家长的理解门槛。
- 状态文案要像“上课安排提示”，不像系统报错或后台状态码。
- 首页主标题遵循原型使用 `我的课堂`；导航项使用 `主页`、`设置`。
- 推荐课次相关文案优先使用明确时间表达，例如“09:55 开始入场”或“10:00 正式开课”。
- 锁定状态统一使用“已开课，停止入场”，不使用“已失效”“不可用”“权限不足”。
- 下一节课提示必须出现具体时间，不写模糊表达如“稍后开始”。
- 原型中的 `上课` 按钮可在实现中统一成 `进入课堂`，以更准确表达入场窗口和路由行为。
- 即使正文使用中文，整体 copy 的节奏仍应参考 `DESIGN.md` 与 Stitch 原型：短、准、直接，不堆叠解释性文案。
